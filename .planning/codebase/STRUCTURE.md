**tests/:**
- Purpose: Automated test suite
- Contains: Unit tests, integration tests, mock fixtures

**.planning/codebase/:**

- Generated: Yes (created by GSD mapper)
- Committed: Yes (these are reference docs)

## Naming Conventions

**Files:**
- `main.py` - CLI and orchestration
- `models.py` - Pydantic BaseModel definitions
- `protocols.py` - Protocol (structural typing) definitions
- `config.py` - Module-level constants
- `[component].py` - One file per major component (scorer, optimizer, futgg_client)
- `test_[component].py` - One test file per component tested
- `mock_[component].py` - Mock implementations for testing

**Directories:**
- `src/` - Source code (main application)
- `tests/` - Test code (mirrors src structure)
- `.planning/codebase/` - Documentation generated by GSD

**Functions:**
- `snake_case` - All functions use snake_case
- `async def` - Async functions prefixed with `async` (e.g., `async def discover_players()`)
- Private helpers - Prefixed with `_` (e.g., `_get_price_at_time()`)

**Classes:**
- `PascalCase` - All classes use PascalCase (e.g., `FutGGClient`, `PlayerMarketData`)
- Protocol names - Suffixed with "Protocol" (e.g., `MarketDataClient` is a protocol)

**Variables:**
- `snake_case` - Local variables, function parameters, module constants
- `UPPER_CASE` - Module-level constants (e.g., `EA_TAX_RATE`, `TARGET_PLAYER_COUNT`)

**Types:**
- Pydantic models - PascalCase (e.g., `Player`, `SaleRecord`)
- Dict keys - snake_case (e.g., scored dict has `"buy_price"`, `"net_profit"`)

## Where to Add New Code

**New Feature (e.g., a new scoring metric):**
- Primary code: `src/scorer.py` - add logic to `score_player()`, add new dict key to return
- Tests: `tests/test_scorer.py` - add test function for new metric
- Config: `src/config.py` - if new constant needed (e.g., MIN_NEW_METRIC)

**New Component/Module (e.g., alternative data source):**
- Implementation: `src/[component_name].py`
- Interface: Already defined in `src/protocols.py:MarketDataClient` - implement this protocol
- Tests: `tests/test_[component_name].py`
- Integration: Update `src/main.py:run()` to accept optional client parameter (already supports this)

**Utilities (shared helpers):**
- Location: `src/[utility_name].py` or add to existing module if closely related
- Example: Price calculation logic could live in `src/scorer.py` or new `src/pricing.py`
- Tests: `tests/test_[utility_name].py`

**New Display Format (e.g., JSON export):**
- Primary code: `src/main.py` - add function similar to `export_csv(selected)` (lines 133-156)
- Tests: `tests/test_integration.py` - test new export format

**New CLI Option (e.g., --output-format flag):**
- Implementation: `src/main.py` - add `@click.option()` decorator and parameter to `main()`
- Pipeline: Pass to `run()` function
- Tests: `tests/test_integration.py` - test with new option

## Import Patterns

**Standard library imports at top:**
```python
from datetime import datetime
import asyncio
import logging
```

**Third-party imports after stdlib:**
```python
import httpx
import click
from pydantic import BaseModel
```

**Local imports last:**
```python
from src.models import Player, PlayerMarketData
from src.protocols import MarketDataClient
```

**No circular imports:**
- `src/models.py` has no imports from other src modules (only stdlib, pydantic)
- `src/protocols.py` imports only `src/models.py`
- `src/config.py` has no imports from other src modules
- `src/futgg_client.py` imports models and config, no business logic

**Relative imports used in tests:**
- `from tests.mock_client import make_player`
- `from src.scorer import score_player`
