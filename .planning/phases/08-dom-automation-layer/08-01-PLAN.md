  <action>
Create `src/server/api/automation.py` with a FastAPI APIRouter (prefix="/api/v1") containing three endpoints:

**1. GET /api/v1/automation/daily-cap**
Returns JSON: `{ "count": int, "cap": int, "capped": bool, "date": "YYYY-MM-DD" }`
- Query the `daily_transaction_count` table for today's date (UTC).
- If no row exists for today, return count=0, cap=500, capped=false.
- `capped` = count >= cap.

**2. POST /api/v1/automation/daily-cap/increment**
Increments today's counter by 1. Returns same shape as GET.
- Upsert into `daily_transaction_count`: if row for today exists, increment count; if not, insert with count=1.
- Default cap = 500. Cap is stored per-row so user can configure it.
- Use PostgreSQL `INSERT ... ON CONFLICT (date) DO UPDATE SET count = daily_transaction_count.count + 1`.

**3. GET /api/v1/portfolio/player-price/{ea_id}**
Returns JSON: `{ "ea_id": int, "buy_price": int, "sell_price": int }`
- Query `portfolio_slots` for the given ea_id.
- If not found, return 404 with `{"detail": "Player not in portfolio"}`.
- This endpoint goes on the automation router (not portfolio router) to keep it grouped with automation concerns.
