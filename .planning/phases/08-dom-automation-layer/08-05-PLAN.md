Create `extension/src/automation-loop.ts` â€” the main orchestrator that drives the continuous buy/list/relist cycle per D-02.

**Import pattern:**
```typescript
import { AutomationEngine, AutomationError, jitter } from './automation';
import { executeBuyCycle, type BuyCycleResult } from './buy-cycle';
import { executeTransferListCycle, scanTransferList, type TransferListCycleResult } from './transfer-list-cycle';
import type { ActionNeeded, ExtensionMessage } from './messages';
```

**Main export: `runAutomationLoop(engine: AutomationEngine, sendMessage: (msg: any) => Promise<any>): Promise<void>`**

This function runs the continuous automation cycle until stopped. It is called by the content script when the user clicks "Start Automation".

**Implementation per D-02 (full cycle):**

```
while (!engine.isStopping) {
  // Phase A: Get actions needed (D-19, D-33)
  // Phase B: Buy cycle for each BUY action (D-02)
  // Phase C: Scan transfer list + relist + clear sold (D-02)
  // Phase D: Handle sold players (rebuy) (D-14)
  // Loop back to Phase A
}
```

**Phase A â€” Get actions needed (cold start / resume per D-19)**
1. Send `{ type: 'ACTIONS_NEEDED_REQUEST' }` via `sendMessage` to get backend's view of what each player needs.
2. Parse the response `ActionsNeededData` â€” extract players with `action === 'BUY'`.
3. Also call `scanTransferList()` to get the DOM's current state (D-18: detect what actually exists).
4. Reconcile: if backend says BUY but DOM shows the card is already listed, skip the buy.

**Phase B â€” Buy all portfolio players (D-02)**
1. Check daily cap first: send `{ type: 'DAILY_CAP_REQUEST' }`. If capped, skip to Phase C (D-24, AUTO-04).
2. Set engine state: `engine.setState('BUYING', 'Starting buy cycle')`.
3. For each player needing a BUY action:
   - If `engine.isStopping`, break (D-17: graceful stop between actions).
   - Fetch fresh price: send `{ type: 'FRESH_PRICE_REQUEST', ea_id: player.ea_id }`. Update player's buy_price and sell_price with fresh values (D-13, D-31).
   - Set status: `engine.setState('BUYING', \`Buying: ${player.name}\`)`.
   - Call `executeBuyCycle(player, sendMessage)`.
   - Handle result:
     - `bought`: `engine.setLastEvent(\`Bought ${player.name} for ${result.buyPrice}\`)`. Send trade report: `{ type: 'TRADE_REPORT', ea_id: player.ea_id, outcome: 'bought', price: result.buyPrice }`. Then send list report: `{ type: 'TRADE_REPORT', ea_id: player.ea_id, outcome: 'listed', price: player.sell_price }`.
     - `skipped`: `engine.setLastEvent(\`Skipped ${player.name}: ${result.reason}\`)`.
     - `error`: Check if it's an AutomationError (D-22, D-23). If so, call `engine.setError(result.reason)` and return (stop the loop).
   - Increment daily cap: send `{ type: 'DAILY_CAP_INCREMENT' }`. Check if now capped â€” if so, log and break out of buy loop.

**Phase C â€” Scan transfer list + relist + clear sold (D-02, D-03)**
1. If `engine.isStopping`, return.
2. Set state: `engine.setState('SCANNING', 'Scanning transfer list')`.
3. Call `executeTransferListCycle(sendMessage)`.
4. Handle result:
   - If `relistedCount > 0`: `engine.setLastEvent(\`Relisted ${result.relistedCount} cards\`)`.
   - Set state to RELISTING during relist, then back to SCANNING.
   - Track sold players for rebuy in Phase D.

**Phase D â€” Handle sold players â€” rebuy (D-14, D-15)**
1. For each sold card detected in Phase C:
   - Match the sold `DetectedItem` to a portfolio player by name + rating (DOM names may be short per CONTEXT.md â€” use `endsWith` matching from trade observer pattern).
   - If matched and not capped: fetch fresh price (D-14), then run `executeBuyCycle` for that player.
   - Report the sale outcome: `{ type: 'TRADE_REPORT', ea_id: matched.ea_id, outcome: 'sold', price: soldItem.binPrice }`.
   - Track profit: `engine.addProfit(soldItem.binPrice - matched.buy_price)` (approximate â€” EA tax applied backend-side).

**Edge case handling:**
- **D-35 (out of coins):** If `executeBuyCycle` returns `{ outcome: 'error', reason }` and the reason indicates insufficient coins (detect from DOM â€” look for a coins-related error element), switch to relist-only mode for the rest of this cycle iteration. Log: "Out of coins â€” relist-only mode".
- **D-36 (transfer list full):** If buy succeeds but list fails because transfer list is full, log warning and continue. The card stays in unassigned pile â€” next cycle will detect and list it.
- **D-38 (session expiry):** After any AutomationError, check if the page has redirected to login (check URL or login page DOM element). If so, set error: "EA session expired â€” please log in and restart automation".
- **D-34 (leftovers):** Leftover players (is_leftover=true in actions-needed) are handled passively by the transfer list scan â€” Relist All relists everything including leftovers. No buy actions for leftovers.

**Between each full cycle iteration:** Add a longer pause (jitter with 5000-10000ms) before starting the next cycle to avoid hammering the EA servers.

**Error boundary:** Wrap the entire loop body in try/catch. If an `AutomationError` is caught, call `engine.setError(error.message)` and return. For any other error, `engine.setError('Unexpected error: ' + error.message)` and return. The UI will show the error state.
