**Import pattern:**
```typescript
import * as SELECTORS from './selectors';
import { requireElement, clickElement, waitForElement, jitter, AutomationError } from './automation';
import { navigateToTransferList, isOnTransferListPage } from './navigation';
import { readTransferList, type DetectedItem } from './trade-observer';
import type { ActionNeeded } from './messages';
```

**Export types:**
```typescript
export type TransferListScanResult = {
  listed: DetectedItem[];
  expired: DetectedItem[];
  sold: DetectedItem[];
};

export type TransferListCycleResult = {
  scanned: TransferListScanResult;
  relistedCount: number;
  soldCleared: number;
  isCapped: boolean;
};
```

**Main function: `executeTransferListCycle(sendMessage: (msg: any) => Promise<any>): Promise<TransferListCycleResult>`**

**Step 1 — Navigate to transfer list (D-04)**
If not already on the transfer list page (`isOnTransferListPage()` returns false), call `navigateToTransferList()`.

**Step 2 — Scan all pages of the transfer list (D-39)**
Implement `scanAllPages(): Promise<TransferListScanResult>`:
- Call `readTransferList()` to get items on current page.
- Categorize each item by its `status` field into `listed`, `expired`, `sold` arrays.
- Check if a pagination "next" button exists and is clickable (not disabled). Use `document.querySelector(SELECTORS.PAGINATION_NEXT_BUTTON)` — if it exists and is not disabled, click it with `clickElement`, wait for the page content to update (jitter + poll for new items), then scan again.
- Repeat until no more "next" pages.
- Return the accumulated `TransferListScanResult`.

**Step 3 — Check daily cap (D-24, AUTO-04)**
Send `{ type: 'DAILY_CAP_REQUEST' }` via `sendMessage` to get current cap status.
If response `capped === true`, set `isCapped = true`. This affects the main cycle loop (Plan 05 will skip buys when capped).

**Step 4 — Relist expired cards (D-03, AUTO-03)**
If there are expired items in the scan:
- Navigate back to the first page of the transfer list if paginated (click "prev" until no more prev button, or re-navigate).
- Find and click the `RELIST_ALL_BUTTON` using `requireElement` and `clickElement`.
- Wait for confirmation/completion (jitter + poll for the button to disappear or expired items to clear).
- The Relist All button relists at the same price cards were originally listed at — this is EA's built-in behavior, aligned with D-03 (locked OP price).
- Count the number of relisted items = `expired.length`.

**Step 5 — Report relist outcomes to backend**
For each expired card that was relisted, send a trade report via `sendMessage`:
```typescript
sendMessage({
  type: 'TRADE_REPORT_BATCH',
  records: expired.map(item => ({
    ea_id: 0, // Will need to match by name — see note below
    outcome: 'relisted',
    price: item.binPrice,
  })),
});
```
**Note on ea_id matching:** `DetectedItem` does not contain `ea_id`. The main automation loop (Plan 05) will need to match detected items to portfolio players by name+rating. For this module, export the raw scan results and let the caller handle ea_id resolution.

**Step 6 — Clear sold cards (D-02)**
If there are sold items in the scan:
- Find and click the `CLEAR_SOLD_BUTTON` using `requireElement` and `clickElement`.
- Wait for confirmation (jitter).
- Count cleared = `sold.length`.

**Step 7 — Return result**
```typescript
return {
  scanned: { listed, expired, sold },
  relistedCount: expired.length,
  soldCleared: sold.length,
  isCapped,
};
```

**Also export a standalone scan function for resume/cold-start (D-18, D-19):**
```typescript
export async function scanTransferList(): Promise<TransferListScanResult>
```
This calls `navigateToTransferList()` then `scanAllPages()` — useful for the resume flow where the cycle needs to assess current state without relisting.

**Error handling:**
- If `RELIST_ALL_BUTTON` is not found but expired items exist, log a warning but do not throw (it may mean there are no expired items on the current page view).
- If `CLEAR_SOLD_BUTTON` is not found but sold items exist, log a warning.
- Let `AutomationError` from `requireElement` propagate for critical navigation failures.
