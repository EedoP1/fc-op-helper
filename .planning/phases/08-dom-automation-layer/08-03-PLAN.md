
**Main function: `executeBuyCycle(player: ActionNeeded, sendMessage: (msg: any) => Promise<any>): Promise<BuyCycleResult>`**

Implementation per locked decisions:

**Step 1 — Navigate to search page (D-04)**
If not already on the search page (`isOnSearchPage()` returns false), call `navigateToTransferMarket()`.

**Step 2 — Fill search fields (D-05)**
- Clear and type player name into `SEARCH_PLAYER_NAME_INPUT` using `typePrice` (reuse digit-by-digit typing for text too — or use a simpler `typeText` helper that sets value + dispatches input event for text fields).
  - For player name typing: set `input.value = player.name`, then dispatch `input` and `change` events. Player name search in EA is a typeahead — after typing, wait 1-2s for suggestions dropdown, then click the matching suggestion. Use `waitForElement` to find a suggestion list item whose text includes the player name.
- If player has a rarity, select it from the quality dropdown (D-05). Use `clickElement` on the dropdown, then click the matching option.

**Step 3 — Price discovery via binary search (D-06, D-07, D-08, D-09)**
- Set max BIN to `player.buy_price` (target buy price).
- Set min BID to a varying value (start at 0; change by +50/-50 on each retry to bust EA's cache per D-09).
- Click search button and wait for results.
- **If no results found:** Step max BIN up by 5% increments. If max BIN exceeds `player.buy_price * 1.05`, skip this player (price guard D-08). Return `{ outcome: 'skipped', reason: 'Price above guard' }`.
- **If results found:** The cheapest card is the first result. Read its BIN price from the DOM (use `ITEM_BIN_PRICE` selector on the first `SEARCH_RESULT_ITEM`).
  - If BIN > `player.buy_price * 1.05`: skip. Return `{ outcome: 'skipped', reason: 'Cheapest card above price guard' }`.
  - If BIN <= `player.buy_price * 1.05`: proceed to buy (D-07: buy even if below target).

**Step 4 — Execute Buy Now (D-26, D-10, D-11)**
- Click the first search result item to select it.
- Click `BUY_NOW_BUTTON`.
- Wait for `BUY_CONFIRM_BUTTON` to appear (buy confirmation dialog).
- Click `BUY_CONFIRM_BUTTON`.
- Wait briefly (jitter) and check for success:
  - If `BUY_EXPIRED_INDICATOR` appears: card was sniped. Increment retry counter. If retries < 3 (D-10), go back to Step 3 (re-search). If retries >= 3 (D-11), return `{ outcome: 'skipped', reason: 'Sniped 3 times' }`.
  - If no expired indicator: buy succeeded. Record the actual BIN price paid.

**Step 5 — List immediately at locked OP price (D-12, D-13)**
- After successful buy, the post-buy screen should show a "List on Transfer Market" option.
- Click `LIST_ON_MARKET_BUTTON`. Wait for listing dialog to appear.
- Type `player.sell_price` into `LIST_BIN_PRICE_INPUT` using `typePrice`.
- Type a start price into `LIST_START_PRICE_INPUT` — use `player.sell_price - 100` (or `player.sell_price` if you prefer; EA requires start price <= BIN).
- Click `LIST_CONFIRM_BUTTON`.
- Wait for confirmation (jitter).
- Return `{ outcome: 'bought', buyPrice: actualBinPaid }`.

**Step 6 — Increment daily cap**
After each search attempt (not just buys), send `{ type: 'DAILY_CAP_INCREMENT' }` via `sendMessage`. This tracks searches+buys per D-24.

**Error handling:**
- Wrap the entire function in try/catch. If `AutomationError` is thrown, return `{ outcome: 'error', reason: error.message }`.
- Any unexpected error: return `{ outcome: 'error', reason: 'Unexpected: ' + error.message }`.

**Important implementation notes:**
- Use `jitter()` between every DOM interaction for human-paced timing (D-28, AUTO-05).
- The min BID cache-busting value (D-09) should use a module-level counter that increments by 50 each search to ensure the value changes.
- Use actual selector constant names from selectors.ts. Read that file first and adapt if Plan 01 used different names than the placeholders listed above.
