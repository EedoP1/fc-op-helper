    Create `src/server/listing_tracker.py` with three public async functions:

    **1. `async def record_listings(ea_id, live_auctions_raw, current_lowest_bin, completed_sales, session) -> dict`**

    Per D-01, D-04, D-07, D-08:
    - For each entry in `live_auctions_raw` (list of dicts from fut.gg liveAuctions):
      - Build fingerprint: check if entry has "tradeId" or "id" field. If yes: `f"{ea_id}:{entry['tradeId']}"`. If no: fall back to `f"{ea_id}:{entry['buyNowPrice']}:{bucket}"` where bucket = `first_seen_at` rounded to 10-minute windows.
      - Upsert into ListingObservation using `sqlite_insert().on_conflict_do_update()` on fingerprint:
        - On INSERT: set all fields, `first_seen_at=now`, `last_seen_at=now`, `scan_count=1`, `outcome=None`
        - On CONFLICT: update `last_seen_at=now`, `scan_count=scan_count+1`
      - `market_price_at_obs = current_lowest_bin`
    - Return `{"recorded": count, "fingerprints": list_of_fps}` for downstream use.

    **2. `async def resolve_outcomes(ea_id, current_fingerprints, completed_sales, session) -> dict`**

    Per D-02, pitfall 2:
    - Query all ListingObservation rows for this ea_id where `outcome IS NULL` and `fingerprint NOT IN current_fingerprints` (i.e., listings that disappeared).
    - Group disappeared listings by buy_now_price.
    - For each price group:
      - Count matching sales from `completed_sales` (list of SaleRecord) where `sold_price == price` and `sold_at` is between the listing's `first_seen_at` and now (with 1-hour buffer).
      - Proportional assignment: first M listings marked "sold", remaining N-M marked "expired" (M = number of matching sales, N = number of disappeared listings at that price).
    - Set `resolved_at = datetime.utcnow()` on all resolved rows.
    - Return `{"sold": sold_count, "expired": expired_count}`.

    **3. `async def aggregate_daily_summaries(ea_id, target_date, session) -> int`**

    Per D-13, D-14:
    - Query all resolved ListingObservation rows for `ea_id` where `first_seen_at` is on `target_date` (YYYY-MM-DD).
    - For each margin in MARGINS (imported from src.scorer):
      - Count listings where `buy_now_price >= market_price_at_obs * (1 + margin/100)` → these are OP listings at this margin.
      - Split OP listings into sold vs expired by outcome.
      - Count total listings (regardless of OP status).
    - Upsert one DailyListingSummary row per (ea_id, date, margin_pct) with counts.
    - Return number of summary rows written.
