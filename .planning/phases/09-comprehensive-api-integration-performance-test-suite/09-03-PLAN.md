**Tests to write:**

1. `test_portfolio_slots_unique_constraint`:
   - Seed slot with ea_id=X
   - Seed slot with same ea_id=X again (POST /portfolio/slots)
   - Query DB: SELECT COUNT(*) FROM portfolio_slots WHERE ea_id=X -> assert 1 (upsert, not duplicate)

2. `test_confirm_clean_slate_actually_deletes`:
   - Seed 3 slots via POST /portfolio/slots
   - POST /portfolio/confirm with only 1 player
   - Query DB: SELECT COUNT(*) FROM portfolio_slots -> assert 1 (clean slate delete)

3. `test_delete_preserves_trade_records`:
   - Seed slot, complete a BUY action (creates TradeRecord)
   - DELETE /portfolio/{ea_id}
   - Query DB: SELECT COUNT(*) FROM trade_records WHERE ea_id=X -> assert >= 1 (records preserved per D-DELETE)
   - Query DB: SELECT COUNT(*) FROM portfolio_slots WHERE ea_id=X -> assert 0 (slot deleted)

4. `test_delete_cancels_pending_actions`:
   - Seed slot, GET /actions/pending (creates IN_PROGRESS action)
   - DELETE /portfolio/{ea_id}
   - Query DB: SELECT status FROM trade_actions WHERE ea_id=X -> assert all are "CANCELLED"

5. `test_stale_action_reset`:
   - Seed slot
   - GET /actions/pending -> action claimed (IN_PROGRESS)
   - Wait or manipulate: the stale timeout is 5 minutes. Instead of waiting, directly verify the stale reset logic by:
     a) Query DB to get the action's claimed_at
     b) Update claimed_at to 6 minutes ago via direct DB update
     c) GET /actions/pending -> should reset the stale action and return it again
   - Assert the action is returned (was reset from stale IN_PROGRESS back to PENDING)

6. `test_no_action_created_when_listed`:
   - Seed slot, BUY -> bought, LIST -> listed
   - Query DB: GET /actions/pending -> assert action is None
   - Query DB: SELECT COUNT(*) FROM trade_actions WHERE ea_id=X AND status='PENDING' -> assert 0
   - The "listed" state means card is on market, no action needed

7. `test_trade_record_id_monotonic`:
   - Seed slot, create 5 trade records in sequence (bought, listed, sold, bought, listed)
   - Query DB: SELECT id FROM trade_records WHERE ea_id=X ORDER BY id
   - Assert ids are strictly monotonically increasing

8. `test_cleanup_fixture_works` (D-15):
   - Seed slot, create trade record
   - This test just verifies the data exists
   - The NEXT test (test_cleanup_verified) verifies the cleanup fixture deleted it
   Note: pytest test ordering isn't guaranteed. Instead, verify cleanup works by:
   - Checking that at the START of this test, portfolio_slots is empty (cleanup from previous test ran)
   - Then seed data for this test's assertions

9. `test_portfolio_generate_no_duplicate_ea_ids` (D-07):
   - POST /portfolio/generate with budget=500000 (large budget = more players)
   - Extract all ea_ids from response
   - Assert len(ea_ids) == len(set(ea_ids)) (no duplicates in generated portfolio)

10. `test_portfolio_confirm_preserves_prices`:
    - POST /portfolio/generate, pick 3 players
    - POST /portfolio/confirm with those 3 players (include buy_price, sell_price)
    - GET /portfolio/confirmed
    - Assert each player's buy_price and sell_price match what was confirmed

11. `test_batch_records_single_commit`:
