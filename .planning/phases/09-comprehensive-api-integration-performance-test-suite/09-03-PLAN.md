<!-- Action lifecycle state machine (from actions.py _derive_next_action): -->
Portfolio slot lifecycle:
  No records        -> BUY action  (target_price = buy_price)
  "bought" record   -> LIST action (target_price = sell_price)
  "listed" record   -> waiting (no action, card on market)
  "sold" record     -> BUY action  (cycle complete, restart)
  "expired" record  -> RELIST action (target_price = sell_price)

Action flow:
  1. POST /portfolio/slots — seed portfolio
  2. GET /actions/pending — get next action (derived from lifecycle state)
  3. POST /actions/{id}/complete — record outcome
  4. Repeat 2-3 for next lifecycle step

Portfolio status maps outcomes to statuses:
  "bought"  -> "BOUGHT"
  "listed"  -> "LISTED"
  "sold"    -> "SOLD"
  "expired" -> "EXPIRED"
  No record -> "PENDING"

Profit summary:
  - "bought" outcomes sum as total_spent
  - "sold" outcomes sum as total_earned_gross, then * (1 - 0.05) for EA tax
  - net_profit = total_earned - total_spent

Fixtures from conftest.py:
  - client: httpx.AsyncClient (function-scoped)
  - cleanup_tables: autouse, deletes mutable tables after each test
  - base_url: str for direct httpx usage in concurrent tests

    ```python
        """Get pending action, assert type, complete it, return action data."""
        r = await client.get("/api/v1/actions/pending")
        assert r.status_code == 200
        action = r.json()["action"]
        assert action is not None, f"Expected {expected_type} action but got null"
        assert action["action_type"] == expected_type
        r2 = await client.post(f"/api/v1/actions/{action['id']}/complete",
                                json={"price": price, "outcome": outcome})
        assert r2.status_code == 200
        return action
    ```

    Note: These are real HTTP over loopback. Thresholds are generous because this includes network overhead, SQLite file I/O, and Python async overhead. Not in-memory baselines.

    `test_health_latency` — GET /health should respond in < 100ms:
    ```python
    async def test_health_latency(client):
        # Warmup
        await client.get("/api/v1/health")
        # Measure
        times = []
        for _ in range(10):
            start = time.perf_counter()
            r = await client.get("/api/v1/health")
            elapsed = (time.perf_counter() - start) * 1000
            assert r.status_code == 200
            times.append(elapsed)
        p95 = sorted(times)[int(len(times) * 0.95)]
        assert p95 < 100, f"Health p95={p95:.1f}ms exceeds 100ms"
    ```
