## Decisions Made

- jsdom normalizes hex colors to rgb() format — tests assert `rgb(58, 58, 94)` (= `#3a3a5e`) for active tab background
- `renderPortfolioContent(parent)` receives an HTMLElement parent so it can render into the `.op-seller-tab-content` slot rather than directly to container
- `renderDashboard()` locates `.op-seller-tab-content` inside container as its target; renderConfirmed creates this element before calling renderTabContent()

## Deviations from Plan

None - plan executed exactly as written. The only adjustment was making test assertions use jsdom's normalized rgb() color format instead of hex, which is not a deviation from the implementation spec.

## Issues Encountered

jsdom normalizes hex color strings (e.g. `#3a3a5e`, `#f88`) to `rgb()` format when read back via `element.style.color/background`. Two tests initially failed with this mismatch; assertions updated to use `rgb()` values.
