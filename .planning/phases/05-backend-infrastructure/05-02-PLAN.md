**POST /api/v1/portfolio/slots** — seed/update portfolio slots so the action queue has data to work with:
- Request body (Pydantic model `SeedSlotsPayload`): `slots: list[SlotEntry]` where `SlotEntry` has `ea_id: int`, `buy_price: int`, `sell_price: int`, `player_name: str`
- For each entry in slots:
  - Check if a PortfolioSlot with this ea_id already exists
  - If exists: update buy_price, sell_price
  - If not: insert new PortfolioSlot with ea_id, buy_price, sell_price, added_at=utcnow()
- Commit. Return `{"status": "ok", "count": len(slots)}`
- Use status code 201 for successful creation

**GET /api/v1/actions/pending** (per D-01, D-02, D-03, D-04):
1. Reset stale: `UPDATE trade_actions SET status='PENDING', claimed_at=NULL WHERE status='IN_PROGRESS' AND claimed_at < (now - 5 min)`
2. Check for existing PENDING action: `SELECT ... WHERE status='PENDING' ORDER BY created_at LIMIT 1`
3. If found: set status='IN_PROGRESS', claimed_at=now, commit, return it
4. If not found: derive next action from portfolio_slots + trade_records:
   a. Query all portfolio_slots
   b. For each slot, check trade_records to determine lifecycle state:
      - No "bought" record for this ea_id -> BUY action (target_price = slot.buy_price)
      - Has "bought" but no "listed" record -> LIST action (target_price = slot.sell_price)
      - Has "listed" + "expired" (most recent outcome is "expired") -> RELIST action (target_price = slot.sell_price)
      - Has "listed" + "sold" -> complete cycle, skip (or start new BUY cycle)
   c. If an action is derived, INSERT into trade_actions with status='PENDING', then claim it (set IN_PROGRESS + claimed_at)
   d. If no action needed, return `{"action": null}`
5. Response shape: `{"action": {"id": int, "ea_id": int, "action_type": str, "target_price": int, "player_name": str}}`

**POST /api/v1/actions/{action_id}/complete** (per D-05):
- Request body (Pydantic model `CompleteActionPayload`): `price: int`, `outcome: str` (one of "bought", "listed", "sold", "expired")
- Look up TradeAction by id. If not found, return 404.
- Insert TradeRecord with ea_id from action, action_type from action, price and outcome from payload, recorded_at = utcnow()
- Set action.status = "DONE", action.completed_at = utcnow()
- Commit. Return `{"status": "ok", "trade_record_id": record.id}`

**Action derivation detail for lifecycle state detection:**
For a given ea_id, query trade_records ordered by recorded_at DESC. The most recent outcome determines next action:
- No records at all -> BUY
- Most recent is "bought" -> LIST
- Most recent is "listed" -> waiting (no action needed, card is on market)
- Most recent is "sold" -> cycle complete, start new BUY
- Most recent is "expired" -> RELIST
