# Quick Task: Portfolio Leftover Tracking

## Goal
When generating a new portfolio, keep unsold players from the old portfolio as "leftovers" and provide a single unified endpoint that tells the user exactly what to do for every player (BUY/LIST/RELIST + details + price).

## Changes

### Task 1: Add `is_leftover` column to PortfolioSlot
- **File:** `src/server/models_db.py`
- Add `is_leftover: Mapped[bool]` column (default False)

### Task 2: Update confirm endpoint to preserve unsold players
- **File:** `src/server/api/portfolio.py` — `confirm_portfolio()`
- Before clean slate: query old slots + their latest TradeRecord outcome
- Players with last outcome bought/listed/expired → mark `is_leftover=True`, keep slot
- Players with last outcome sold/None (never bought) → delete
- Insert new portfolio slots with `is_leftover=False`
- New portfolio players that overlap with leftovers: promote leftover back to active

### Task 3: Add `/api/v1/portfolio/actions-needed` endpoint
- **File:** `src/server/api/portfolio.py` (new endpoint)
- Returns flat prioritized list for every player across portfolio + leftovers:
  ```json
  {
    "actions": [
      {"ea_id": 123, "name": "Messi", "rating": 91, "position": "RW",
       "action": "BUY", "target_price": 50000, "is_leftover": false,
       "futgg_url": "..."},
      {"ea_id": 456, "name": "Ronaldo", "rating": 90, "position": "ST",
       "action": "LIST", "target_price": 65000, "is_leftover": true,
       "futgg_url": "..."},
    ],
    "summary": {"to_buy": 80, "to_list": 5, "to_relist": 3, "waiting": 12}
  }
  ```
- Priority: LIST > RELIST > BUY (sell what you have first, then buy new)
- Players currently "listed" (on market) excluded from actions (nothing to do)

### Task 4: Update confirmed portfolio endpoint to separate leftovers
- **File:** `src/server/api/portfolio.py` — `get_confirmed_portfolio()`
- Return `{"portfolio": [...], "leftovers": [...], "count": N, "leftover_count": N}`

### Task 5: Update action derivation to respect leftovers
- **File:** `src/server/api/actions.py` — `_derive_next_action()`
- Leftover slots: skip BUY action (never buy a leftover — you already own it)
- If leftover has no trade record: derive LIST (not BUY)

### Task 6: Auto-cleanup leftovers on sale
- **File:** `src/server/api/actions.py` — `complete_action()` and `direct_trade_record()`
- When outcome="sold" and slot.is_leftover=True → delete the PortfolioSlot

### Task 7: Update portfolio status to distinguish leftovers
- **File:** `src/server/api/portfolio_status.py`
- Add `is_leftover` field to each player row in response

### Task 8: Alembic migration for is_leftover column
- Generate migration adding `is_leftover` boolean column with default False
