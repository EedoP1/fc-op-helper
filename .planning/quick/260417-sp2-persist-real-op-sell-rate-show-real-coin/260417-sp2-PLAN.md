<objective>
Plumb the real OP success rate and a real coins/hr metric from the scorer through persistence to the CLI. Three misnamed columns in PlayerScore currently store composite or zero values and are displayed to the user as if they were real rates and coin amounts; this plan replaces those display values with ground truth while preserving the optimizer's existing ranking behavior.

Purpose: The user sees honest numbers. `Win%` is actually the OP sell success rate (0-100%), `EP/hr` is actually coins per hour, and `OP Sales` shows real observation counts. The composite ranking score that drives portfolio selection stays exactly where it is (the `expected_profit_per_hour` column / optimizer input) — only the display changes.

Output: Scorer exposes three new fields, scanner persists them into existing columns (no schema change), API computes and returns `coins_per_hour`, CLI reads and displays real values.
</objective>
