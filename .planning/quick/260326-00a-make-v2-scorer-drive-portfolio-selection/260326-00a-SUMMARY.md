# Quick Task 260326-00a: Make v2 Scorer Drive Portfolio Selection — Summary

**One-liner:** v2 scorer's `expected_profit_per_hour` now drives portfolio ranking — optimizer uses `epph/buy_price` as efficiency for v2 players with `ep/buy_price` as fallback for v1, and the portfolio API surfaces `ranking_metric` and `scorer_mix` per response.

## Objective

When a player has `expected_profit_per_hour` from scorer_v2, use that as the ranking metric in `optimize_portfolio()` instead of v1's `expected_profit`. Fall back to v1 for players still bootstrapping (no v2 score).

## Tasks Completed

### Task 1: Update optimize_portfolio to rank by expected_profit_per_hour when available

**Commit:** `05653d8` (feat), `b894f7b` (test/RED)

**Changes in `src/optimizer.py`:**
- Efficiency computation now checks `s.get("expected_profit_per_hour")`: if present and > 0, uses `epph / buy_price`; otherwise falls back to `expected_profit / buy_price`
- Added `_ranking_profit` internal key per entry (epph for v2, ep for v1) used by swap loop comparisons instead of hardcoded `expected_profit`
- Final output sorted by `efficiency` (normalised, comparable across v1/v2) rather than raw `expected_profit`
- `_ranking_profit` stripped from each entry before returning

**Changes in `tests/test_optimizer.py`:**
- Updated `_make_scored` helper to accept optional `expected_profit_per_hour` parameter
- Added `test_v2_player_ranks_by_expected_profit_per_hour`: v2 player with eff=0.25 (via epph) beats v1 player with eff=0.01
- Added `test_v1_fallback_when_no_epph`: two v1 players, higher ep/buy_price wins
- Added `test_mixed_v1_v2_portfolio`: mixed list respects budget, no duplicates, all entries have `expected_profit`

### Task 2: Update portfolio endpoint response to surface ranking metric used

**Commit:** `c716e68`

**Changes in `src/server/api/portfolio.py`:**
- Each player dict now includes `ranking_metric`: `"expected_profit_per_hour"` if epph is present, `"expected_profit"` otherwise
- Added top-level `scorer_mix` to response: `{"v1": N, "v2": N}` summary of how many selected players came from each scorer version

**Changes in `tests/test_portfolio.py`:**
- Added `test_portfolio_returns_scorer_mix`: seeded v1-only app, verifies `scorer_mix` key present with `v2=0` and `v1=count`
- Added `test_portfolio_v2_scored_player`: seeds a `PlayerScore` with `expected_profit_per_hour=450.0` and `scorer_version="v2"`, verifies `ranking_metric="expected_profit_per_hour"` and `scorer_mix.v2=1`
