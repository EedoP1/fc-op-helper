# Quick Task 260326-0d0: Remove Tier-Based Scanning, Replace with Listing-Expiry Scheduling

**One-liner:** Replaced hot/normal/cold tier scheduling and adaptive velocity comparison with pure listing-expiry timing — scan next occurs just before the longest listing under 60min expires, falling back to 56min default.

## What Changed

### src/config.py
- Removed: `SCAN_INTERVAL_HOT`, `SCAN_INTERVAL_NORMAL`, `SCAN_INTERVAL_COLD`, `TIER_PROFIT_THRESHOLD`, `ADAPTIVE_CHANGE_THRESHOLD`, `ADAPTIVE_MIN_INTERVAL_SECONDS`
- Added: `DEFAULT_SCAN_INTERVAL_SECONDS = 3360` (56 minutes = 60min - LISTING_SCAN_BUFFER_SECONDS)
- Kept: `STALE_THRESHOLD_HOURS`, `LISTING_SCAN_BUFFER_SECONDS` (still used)

### src/server/scanner.py
- Removed: `_classify_tier` method entirely
- Rewrote: `_classify_and_schedule` — now uses pure expiry logic only
  - Iterates live_auctions_raw, collects remaining times under 60min
  - Takes max of those (catches the longest about-to-expire listing)
  - Subtracts LISTING_SCAN_BUFFER_SECONDS (4min), floors at 60s
  - Falls back to DEFAULT_SCAN_INTERVAL_SECONDS when no listing under 60min
  - No longer writes `scan_tier` on PlayerRecord
  - No longer does DB query for previous PlayerScore (removed adaptive velocity comparison)
- Updated: `run_bootstrap` — scan_tier now writes `""` instead of `"normal"`
- Updated: `run_discovery` — removed `scan_tier = "cold"` for missing players; only sets `next_scan_at = far_future`
- Updated imports: removed all tier/adaptive constants

### tests/test_scanner.py
- Removed 9 tests: 5 tier classification tests + 4 adaptive scheduling tests
- Renamed test 21 to `test_expiry_based_scheduling_uses_max_under_60min`
  - Updated to use two auctions (30min and 50min), asserts interval based on 50min (max)
- Added `test_expiry_scheduling_defaults_when_no_listing_under_60min` — 90min auction triggers default
- Added `test_expiry_scheduling_no_auctions_uses_default` — None input triggers default
- Updated imports: removed TIER_PROFIT_THRESHOLD, SCAN_INTERVAL_NORMAL, SCAN_INTERVAL_HOT, ADAPTIVE_MIN_INTERVAL_SECONDS; added DEFAULT_SCAN_INTERVAL_SECONDS
