# Quick Task 260325-v54: Improve Scorer to Produce More Optimal Picks

Scorer now evaluates ALL margins and picks the one maximizing expected_profit (net_profit * op_ratio) instead of greedily stopping at the highest margin with 3+ OP sales.

## What Changed

### Problem

The scorer used a greedy-highest approach: iterate margins from 40% down to 3%, return the first with 3+ OP sales. This missed better strategies where a lower margin with many more OP sales produces higher expected_profit.

Example: A player with 3 OP sales at 40% (expected=198) and 40 OP sales at 8% (expected=208) would incorrectly pick 40%.

### Solution

Replaced the early-return loop with an evaluate-all approach that tracks the best expected_profit across all viable margins and returns the optimal one.

**src/scorer.py:**
- Replaced `for margin_pct in MARGINS: ... return` with `best = None; best_expected = 0.0; for margin_pct in MARGINS: ... if expected_profit > best_expected: best = {...}; return best`
- Updated module docstring and function docstring to reflect new behavior

**tests/test_scorer.py:**
- Added `test_picks_margin_maximizing_expected_profit`: Constructs player with 3 sales at 40% margin and 37 at 8% margin, proves scorer picks 8% (expected=208 > 198)
- Added `test_optimal_margin_still_picks_highest_when_it_wins`: Confirms no regression when highest margin genuinely has the best expected_profit
