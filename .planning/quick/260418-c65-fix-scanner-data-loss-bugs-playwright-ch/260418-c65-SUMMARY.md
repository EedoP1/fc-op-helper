# Quick 260418-c65: Fix scanner data-loss bugs (Playwright + futgg + discovery) Summary

Three silent data-loss bugs in the scanner pipeline are fixed: Playwright now polls the Cloudflare challenge for up to 30s instead of guessing once at 5s, futgg_client raises a typed PricesFetchError when prices fail (so tenacity retries instead of silently logging success), and scanner_discovery resets cold-marked players' next_scan_at when they re-appear in the discovery range.

## What Changed

### Task 1 — Playwright Cloudflare poll loop + timeout bumps (commits 87a75db7, 70dc276e)

**RED:** `tests/test_playwright_client.py` (3 tests):
- `test_fetch_prices_polls_and_succeeds` — fake page returns "Just a moment" 3 times then clean JSON; asserts result is parsed and 3-4 polls fired at 0.5s intervals
- `test_fetch_prices_times_out_returns_none` — fake page always returns challenge; monkeypatch `time.monotonic` to advance 0.5s per `asyncio.sleep` call so the test runs in ms; asserts None and fake clock past 30s
- `test_fetch_prices_no_challenge` — happy path, zero sleeps

**GREEN:** `src/server/playwright_client.py`:
- Added `import time`
- Replaced single `await asyncio.sleep(5)` with a `while time.monotonic() < deadline` loop polling every 0.5s for up to 30s
- Bumped `page.goto(..., timeout=30000)` (was 15000)
- Bumped `future.result(timeout=45)` in `get_prices_sync` (was 30) so the outer sync bridge never cuts off the inner 30s poll
- `_resolve_challenge` (startup-only) untouched per plan scope

### Task 2 — PricesFetchError + scanner retry predicate (commits aec7c699, c6c881f3)

**RED:** `tests/test_futgg_client.py` (6 tests across async + sync paths):
- defn ok + prices None → `PricesFetchError`
- both None → returns None (full outage; unchanged)
- both ok but no current_bin → returns None (card not on market; no raise)

**GREEN:**
- `src/futgg_client.py` — new `PricesFetchError(Exception)` class at module top; both `get_player_market_data` (async) and `get_player_market_data_sync` (sync) split the previous `if not defn or not prices: return None` into the three-way matrix described above
- `src/server/scanner.py` — added `PricesFetchError` to `from src.futgg_client import ...` and to the `retry_if_exception_type(...)` tuple on the `_fetch_with_retry` decorator. Now tenacity's 3-attempt exponential backoff retries prices failures instead of silently logging success.

### Task 3 — Cold-mark reset on rediscovery (commits a63d8b81, c9dc8caa)

**RED:** `tests/test_scanner_discovery.py` (3 tests, real in-memory aiosqlite + session_factory):
- Cold player (next_scan_at = now + 24h) re-appearing in discovery → reset to ~now
- Normal-schedule player (next_scan_at = now + 3min) re-appearing → untouched
- Cold player NOT in discovery → stays cold

**GREEN:** `src/server/scanner_discovery.py`:
- Added `update` to the `sqlalchemy` import
- Inserted a single Core UPDATE between the upsert loop and the mark-cold loop:
  ```python
  if discovered_ids:
      reset_threshold = now + timedelta(hours=1)
      await session.execute(
          update(PlayerRecord)
          .where(
              PlayerRecord.ea_id.in_(discovered_ids),
              PlayerRecord.next_scan_at > reset_threshold,
          )
          .values(next_scan_at=now)
      )
  ```
- Mark-cold loop unchanged
